What is D3?

D3 is...

A Javascript library created by Mike Bostok for manipulating documents based on data.

D3 allows you to bind arbitrary data to the DOM, and then apply data-driven transformations to the document.

What does this all mean and why should you even care?

It can be used to create interactive visualizations but can also be a key technology used in visual storytelling.

D3's Strengths

What Are D3's Strengths?

The ability to make api calls based on certain data types such as:

  • json or csv

Provides methods for computing basic summary statistics such as:

  • d3.min, d3.max, d3.extent

Additional methods for grouping data:

  • d3.nest

It comes with predefined layouts, projections, generators, ect.

  • d3.geo

But one of it's major strengths is using Data Binding and the Enter\Update\Exit methods to keep track of the state of the data....more on this later...

React's Strengths

What Are React's Strengths?

Breaking down an app into components.

  • TicTacToe

Standard lifecycle methods are used to control the flow of code execution.

  • ComponentDidMount or ComponentDidUpdate

Animation Lifecycle methods can be use used to emulate D3's Enter\Update\Exit

  • ComponentWillEnter or ComponentWillLeave

But it's real performance gain comes from managing the DOM via the Virutal DOM

Leveraging The Two Technologies

The Best Of Both Worlds...

D3

Using D3's scaling methods such as .ordinal() or .linear()

Using D3's Data Binding and Enter\Update\Exit to add\remove\update data.

React

Break down a data visualization into components.

  • Single Chart Component
  • Multiple Components

Leverage Lifecycle methods to draw and redraw dataviz based on changes to dataset.

  • ComponentDidMount \ ComponentDidUpdate

Incorporate ReactAnimationGroup and emulate D3's Enter\Update\Exit functionality
