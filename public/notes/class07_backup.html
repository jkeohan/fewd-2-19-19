Floats/Clears Review

Floats: Review

• Enable complex layouts by altering a block element's "flow" in the document

• Three rules:

1) Floated elements should be assigned a width

2) Floated elements will shift left or right, until:

⇒ They reach the side of their parent

⇒ They encounter another floating element

3) Floated elements need to be handled using one of the clearing techniques

• Example

Clears: Review

• How we handlefloating elements

• Various approaches to clearing elements:

⇒ clear CSS property

⇒ overflow: hidden;

⇒ clearfix class

The clear CSS Property

• clear is a CSS property that takes:

⇒ left

⇒ right

⇒ both

• Tells non-floating elements where floats are not allowed (left, right, both sides)

Clears: overflow: hidden

• The overflow CSS property determines what happens to content that extends beyond its parent

• Takes the following values: visible, hidden, scroll, auto

• If hidden or auto is applied to an element containing floated elements, will cause the element to extend to accompany any floating children

• More info

Clears: The Clearfix

• Applying overflow may cause undesirable behavior

• Clearfix to the rescue:


	.clearfix:after {
	  content: "";
	  display: block;
	  clear: both;
	}
