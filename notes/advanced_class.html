Data Types

Data Types

• In the machine, there is only data

• The goal of computing is to collect, manage, manipulate, and store this data

• To manipulate data, it's important to know the data type:

⇒ What the data represents

⇒ Possible values for that type

⇒ Operations that can be performed on the data

• ALWAYS KNOW YOUR DATA TYPE!

Data Types

• Numbers

42, 1337, 3.14

• Strings

Single words or sentences surrounded by quotes

'lots of kittens'

• Booleans

Either true or false values

• Arrays

Collections of related data

['manhattan', 'the bronx', 'queens', 'brooklyn', 'staten island']

Arrays

Arrays

• Arrays are a list-like data structure of comma-separated values

• Useful for storing a sequence of related data

• Declare arrays using square brackets: [ ]


	var friends = ['Moe', 'Larry', 'Curly'];

• Element - an item inside the array

• Index - the numerical location of each element in the array (starts at position 0 - aka "0-indexed")

Arrays

• JS provides multiple methods for array creation:

⇒ The new keyword


	var drinks = new Array('beer', 'coffee', 'water');

⇒ Using the array literal


	var drinks = ['beer', 'coffee', 'water'];

• While similar, there are crucial differences

• Use the array literal for now

Arrays

• Use [index] to retrieve values from an array

• Remember that arrays are 0-indexed


	var drinks = ['beer', 'coffee', 'water'];

	drinks[0]
	// 'beer'

	drinks [0+1]
	// 'coffee'

	var x = drinks.length - 1
	drinks[x]
	// 'water'

	drinks[4]
	// undefined

Array Iteration

Array Iteration

• Iteration is the process of running the same function to every element in an array:


  var boroughs = ['manhattan', 'the bronx', 'queens', 'brooklyn', 'staten island'];

  boroughs.forEach(function (borough) {
    console.log(borough)
  })

  // 'manhattan'
  // 'the bronx'
  // 'queens'
  // 'brooklyn'
  // 'staten island'
Array Iteration

• JS arrays have several iteration methods:

.forEach()

.every()

.some()

.filter()

.map()

• Most require a function to be supplied as an argument; this function will be applied to each item in the array, individually

Objects

Objects: Introduction

• So far, we've dealt with simple (primitive) data types:

⇒ String

⇒ Number

⇒ Boolean

⇒ Not so much: undefined, null

• Array, a list-like data-structure (note: complex)

• As our applications grow more powerful, we need to leverage a more complex data-structure, the Object

Objects: Introduction

• Objects are collections of properties, which are key:value pairs:


  {
    name: 'Obama',
    number: 44,
    titles: ['POTUS', 'Prez', 'Commander-In-Chief'],
    speak: function () {
      return 'Pay your taxes.';
    }
  }

• Useful for a simple, structured data store

• Fundamental programming paradigm that helps us structure and categorize our code

• AKA - associative array / hash / dictionary

Objects: Creation

Like arrays, two ways to create objects:

• Object Literal


	var person = {};

⇒ Useful for simple data stores

• Object Constructor


	var person = new Object();

⇒ Useful for creating many instances of a particular type

Objects: Get/Set Values

• Use either dot-notation or bracket notation to set and get propreties


  var person = {
    name: 'Obama'
  };

  // dot-notation
  person.job = 'president';

  person.job;
  // "president"

  // bracket notation (useful if a property has a special character)
  person['bad-habits'] = 'cigarettes';

  person['bad-habits'];
  // "cigarettes"

AJAX

API's: What are they?

• API = Application Programming Interface

• Set of rules/instructions for interacting with a piece of software

• Middleman; allows different pieces of software to "talk"

• API for devices (iPhone); for operating systems (OSX); for JavaScript libraries (jQuery API)

AJAX

• AJAX = Asynchronous JavaScript and XML

• Collection of technologies used to make HTTP (and other asychronous) requests from the browser

• Sends and receives data in a variety of formats, including JSON

• Operates in the "background"; allows us to request/receive data and update the UI without having to refresh the page

jQuery + AJAX

• Vanilla JS is sufficient for making HTTP requests

• It's just verbose...see example

• Also, older browsers handle AJAX calls differently

• jQuery's motto: "Write Less, Do More!"

	$.ajax({
	  url: 'http://your-url.com',
	  type: 'GET',
	  success: function (response) {
	  	console.log(response);
	  },
	})
jQuery + AJAX

• $.ajax() is a method that takes an object of options:


	{
	  url: (the endpoint you want to make the request to),
	  type: (what type of request - GET, POST, PUT, DELETE),
	  success: (function that runs when the request is successful),
	  error: (function that runs when the request fails),
	}
